## ADDED Requirements

### Requirement: CommandMetadata derive macro
The workspace SHALL provide a proc-macro `CommandMetadata` that generates `command_name()` and `command_required_permission()` functions from attributes on the `Command` enum, replacing 105+ manual match arms in `server.rs`.

#### Scenario: Command name derivation
- **WHEN** a `Command` variant is annotated with `#[command(name = "play")]`
- **THEN** the generated `command_name()` function SHALL return `"play"` for that variant

#### Scenario: Command permission derivation
- **WHEN** a `Command` variant is annotated with `#[command(permission = PERMISSION_CONTROL)]`
- **THEN** the generated `command_required_permission()` function SHALL return `PERMISSION_CONTROL` for that variant

#### Scenario: Default permission
- **WHEN** a `Command` variant has no explicit `permission` attribute
- **THEN** the generated code SHALL use `PERMISSION_NONE` as the default permission level

### Requirement: Proc-macro crate structure
The derive macro SHALL live in a new `rmpd-macros` crate within the workspace, following Rust's proc-macro crate requirements.

#### Scenario: Workspace membership
- **WHEN** the workspace `Cargo.toml` is checked
- **THEN** `rmpd-macros` SHALL be listed in the `members` array

#### Scenario: Crate type
- **WHEN** `rmpd-macros/Cargo.toml` is examined
- **THEN** it SHALL specify `[lib] proc-macro = true`

### Requirement: Backward-compatible migration
The derive macro SHALL produce identical behavior to the existing manual `command_name()` and `command_required_permission()` match statements—no command name or permission level SHALL change.

#### Scenario: Equivalence verification
- **WHEN** the derive macro is applied to the `Command` enum
- **THEN** calling `command_name()` and `command_required_permission()` for every variant SHALL return the same values as the current manual implementations

#### Scenario: Compile-time validation
- **WHEN** a `Command` variant is missing the required `#[command(name = "...")]` attribute
- **THEN** the proc-macro SHALL emit a compile-time error with a descriptive message

### Requirement: Existing match arms removed
After the derive macro is applied, the manual `command_name()` and `command_required_permission()` functions in `server.rs` SHALL be removed, replaced by the generated implementations.

#### Scenario: No manual dispatch remaining
- **WHEN** `rmpd-protocol/src/server.rs` is searched for `fn command_name` and `fn command_required_permission`
- **THEN** neither function SHALL exist as manually-written code—they SHALL be generated by the derive macro
